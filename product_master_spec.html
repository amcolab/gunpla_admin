UI Screen Specification – 商品データベース（Product Master App）

対象画面: 商品データベース（Product Master）一覧画面

想定利用者: 倉庫担当者・購買担当・運用担当・管理者（Credie 内部ユーザー）

1. 画面の目的（ビジネス上の役割）

本画面は、Credie 全体の業務においてすべての商品（SKU）の基盤情報を管理する中心画面として機能します。 受注・出荷・在庫・購買・配当ロジックはすべて本マスタのデータに依存します。

1.1 本画面が影響する業務領域

  • 在庫管理（論理在庫・実在庫）
  • Shopify → 受注管理 → 出荷タスク
  • PO（購買）作成・仕入管理
  • 配当・コミッション計算（製品カテゴリ別）
  • レポート・収益分析

2. ユーザーストーリー（利用シナリオ）

本画面が日々どのように利用されるか、実運用の流れに沿って整理します。

2.1 倉庫担当者のシナリオ

  • 入庫・出庫タスクを処理する前に SKU 情報を確認する
  • 商品重量やケース単位を確認し、棚卸や棚移動に利用

2.2 購買担当者のシナリオ

  • PO 作成時に対象商品の仕様・仕入先情報を参照する
  • OEM商品を追加する際に新規登録を行う

2.3 運用チームのシナリオ

  • Shopify 注文に含まれる SKU と Master の SKU をマッチング
  • 受注の不一致（SKU不明、廃盤）を検知する

2.4 管理者のシナリオ

  • 商品ステータスの変更（新商品 / 廃盤）
  • 価格改定の記録管理
  • 商品のライフサイクル管理

3. 主要機能

3.1 商品一覧

  • SKU・商品名・カテゴリー・単位・メーカーなどを表示
  • Kintone 標準リストビューでの表示

3.2 検索 / フィルタ

  • SKU / 商品名 / 型番 / カテゴリー検索に対応
  • フィルタにより目的の商品を迅速に検索

3.3 新規登録（Create）

  • 商品名、SKU、カテゴリー、単位など登録
  • 必須項目は開発時点で確定

3.4 編集（Update）

  • 価格・内容量・メーカー情報など日常的修正

3.5 ステータス管理

  • 販売中 / 販売終了 / 下書きなどの管理

4. 画面レイアウト

5. お客様に確認いただきたい点

  • 画面の目的は業務フローの理解と一致していますか？
  • 表示項目・検索方法は運用に適切ですか？
  • 商品登録・編集フローに不足はありませんか？

6. ビジネス影響（Business Impacts）

  • SKU 情報の誤りは在庫ズレ・誤出荷を招く
  • 仕入情報の誤りは PO ミスとコスト増に直結
  • ステータス管理不備は廃盤品の誤販売につながる
  • 価格情報のズレは利益計算と配当計算に影響

7. 対象範囲 / 非対象範囲（Scope / Out of Scope）

7.1 対象範囲（Scope）

  • 商品マスタデータの登録・更新・閲覧
  • SKU・カテゴリー・価格・メーカー情報の管理
  • 各フロー（受注・在庫・PO）へのデータ連携

7.2 非対象範囲（Out of Scope）

  • 在庫数量の管理（これは Stock App / Physical DB が担当）
  • 受注・出荷タスクの管理（Orders / Task App が担当）
  • Affiliate / Commission の計算（Rewards Ledger が担当）
  • 棚卸ロジック（別の在庫アプリで管理）

8. 開発向けメモ（内部用）

  • CSS は kintone-common-v2.css を使用
  • Kintone 標準画面にカスタムルック＆フィールを適用
  • SKU とステータスは全アプリ横断の整合性が必須
