UI Screen Specification – システム設定（System Setting App）

対象画面: システム設定一覧 & 各種設定詳細画面

想定利用者: Credie 運用チーム・管理者

1. 画面の目的（ビジネス上の役割）

System Setting App は、Credie 業務システム全体の 基幹設定（必須情報）を一元管理するハブです。 これらの設定はシステムの挙動に直接影響するため、正確性と一貫性が重要です。

1.1 管理対象

  • 会社情報（住所・連絡先・ロゴ・請求書情報）
  • 銀行口座情報（Credie 入金用・支払い用）
  • 通知テンプレート（メール本文）
  • 業務ルール（例：税率）
  • SMTP 設定（Credie 送信アカウント）
👉 このアプリは「Credie システムの設定を変更できる唯一の場所」です。

2. 利用者ストーリー（User Scenario）

  • 管理者が「通知テンプレート」を編集し、Partner Portal に送られるメール内容を更新する。
  • 会計担当が「会社情報」や「銀行口座情報」を修正し、請求書や帳票に反映させる。
  • 運用担当が「税率」を更新し、販売・購買ロジックを最新化する。
  • システム管理者が SMTP を更新し、メール送信を安定化させる。

3. 主要機能

  • 設定一覧の表示（List View）
  • 設定項目の編集（Detail / Edit View）
  • 通知テンプレートの編集（テンプレート一覧 & 本文編集）
  • 業務ルール設定（税率等）
  • SMTP メール設定（送信アカウント）
  • 会社情報の管理
  • 銀行口座情報の管理

5. お客様に確認いただきたい点

  • 会社情報に必要な項目はすべて揃っていますか？
  • 通知テンプレートの種類は十分ですか？
  • 税率以外に運用上必要な業務ルールはありませんか？
  • 銀行口座情報画面の項目は正確に運用できますか？

6. 開発向けメモ（参考）

  • 全て 1 App 内の複数 Type（Company / Bank / Notification / Rule / SMTP）のレコードで管理
  • 通知テンプレートは固定・ユーザー追加不可
  • 設定値はミドルウェアがキャッシュして利用
